Eight arms. One terminal. All of GitHub.

Python 3.10+ MIT License v1.1.0 requests rich InquirerPy

Octo is a fully interactive, terminal-native GitHub companion. Browse repositories, preview files with syntax highlighting, clone specific subdirectories, commit and push changes, search code, read issues and pull requests, and inspect contributor stats вҖ” all without leaving your shell.

It talks directly to the GitHub REST API with no dependencies beyond three pure-Python packages. No Electron, no browser tab, no copy-pasting URLs.

v1.1.0 вҖ” Security release
This version fixes a credential injection vulnerability present in v1.0.0. If you are running v1.0.0, upgrade immediately. See the Security section for full details.

в—Ҳ Authentication

Octo works without a token but GitHub limits unauthenticated requests to 60 per hour. A Personal Access Token raises this to 5,000 per hour and unlocks private repositories.

Generate a token

Open GitHub token settings
GitHub вҶ’ Settings вҶ’ Developer Settings вҶ’ Personal Access Tokens вҶ’ Tokens (classic)
Generate new token (classic)
Give it a name like octo-cli so you can identify it later.
Set scopes
Check repo. Add read:org if you want to browse organisation repositories.
Copy the token
You will only be shown it once. Copy it before closing the page.

Add the token to Octo

Main menu  вҶ’  Setup Token

Enter a profile name (e.g. default) and paste your token. Octo validates it against the GitHub API immediately вҖ” the token is only saved if validation succeeds.

Security note
Tokens are stored in ~/.octo_profiles.json with 0600 permissions on Unix. The token never appears in process arguments or shell history. See the Security section for full details.

Binary files and unsupported extensions display the file's GitHub URL instead of attempting a preview.

рҹ”Ҫ Sparse checkout

Navigate to any folder in the file browser and select Clone this path. Octo uses git sparse-checkout to download only that subdirectory вҖ” not the entire repository. This is significantly faster for large monorepos.

Browse Files  вҶ’  navigate into any folder  вҶ’  Clone this path

The clone is saved as ./repo_folder_clone/ in your current working directory. Octo asks if you want to open it in $EDITOR immediately, then offers to start a commit & push session.

New in v1.1.0
After cloning, the file browser now continues running so you can navigate to another folder and clone again without restarting.

рҹ”Қ Code search

Browse  вҶ’  select repo  вҶ’  Search Code

Uses the GitHub Search API scoped to the selected repository. Results list matching file paths. Select any file to load its full content with syntax highlighting.

Rate limits
The Search API is limited to 10 requests/minute unauthenticated and 30/minute with a token. If you hit the limit, Octo will tell you and suggest waiting or adding a token.

рҹ‘Ө Profiles

Octo supports multiple named token profiles so you can switch between GitHub accounts without re-entering tokens.

Storage

Profiles are stored in ~/.octo_profiles.json. The file is created with 0600 permissions on Unix. Writes are atomic вҖ” a crash or interrupt during save cannot leave the file corrupt.

~/.octo_profiles.json
{
  "default": "ghp_...",
  "work":    "ghp_...",
  "oss":     "ghp_..."
}

рҹ”’ Security

v1.1.0 is a dedicated security release. The following issues were identified in v1.0.0 and patched.

High
Shell injection via credential helper вҖ” fixed
The raw token was interpolated into a bash shell function string and passed to git. Tokens containing $, backticks, !, or \ could trigger arbitrary command execution. Fixed by writing the token to a 0600 temp file and passing only the helper script path to git вҖ” the token value never enters any shell evaluation context.
Medium
Token saved before validation вҖ” fixed
In v1.0.0, an invalid token was persisted before being checked. Now the GitHub API is called first; the token is only written to disk on HTTP 200.
Medium
Non-atomic profile writes вҖ” fixed
A crash or Ctrl-C mid-write could leave ~/.octo_profiles.json truncated with no recovery path. Writes now use mkstemp + os.replace(), which is atomic on POSIX and Win32.
Low
Loose permissions not detected вҖ” fixed
The profiles file could be world-readable in Docker or CI environments without any warning. Octo now checks on startup and shows the exact chmod command if the permissions are too broad.
Low
Implicit TLS verification вҖ” fixed
All requests calls now pass verify=True explicitly, making intent clear and preventing environment-variable overrides from silently disabling certificate verification.

Credential security model

When Octo needs to authenticate a git operation (clone, push), it:

Secure credential flow вҖ” v1.1.0
# 1. Token written to a temp file (0600, never in any string interpolation)
token_file = mkstemp()   # e.g. /tmp/.octo_tok_abc123

# 2. A helper script written (0700) that reads from the token file
helper = "#!/bin/sh\necho username=token\ncat TOKEN_FILE\n"

# 3. Git receives only the SCRIPT PATH вҖ” never the token value
git -c credential.helper=HELPER_PATH clone ...

# 4. Both temp files deleted in a finally block вҖ” even on failure
os.unlink(token_file)
os.unlink(helper)
